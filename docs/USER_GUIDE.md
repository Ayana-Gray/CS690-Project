# User Document

## What This App Does
Goal Progress Tracker helps you manage progress across three personal goals and supporting activities:

1. Language Learning
2. Half Marathon Training
3. Novel Creation

It also includes:

- Journal entry tracking
- Goal-specific reminders
- Progress metrics and progress reports

## Starting the Application

1. Open terminal in FinalProject/GoalProgressTracker
2. Run:

```bash
dotnet run
```

3. Wait for the Main Menu prompt.

## Main Menu Navigation

From the Main Menu, select:

1. Language Learning
2. Half Marathon Training
3. Novel Creation
4. Journal
5. Set A Reminder
6. Exit

Type the number for your selection and press Enter.

All Menus are designed to take the user back to the previous menu or to exit the program.

All user prompts are protected from incorrect information from being entered, such as text in numerical prompts or a wrong number being selected amongst specific options. 

Once  a user selects an option in a menu, they are taken to that location. If that location does not prompt for user input, the user can press any key to return to the previous menu.

If a user is prompted to record progress and does not want to, just press enter to return to the previous menu. No progress will be recorded.

## Language Learning

From the Main Menu you are led to another Menu Called Language Learning:

1. View the Language Learning homepage
2. View schedule information
3. Record progress
4. View progress report

From the Language Learning Menu you can view the Language Learning homepage. Which will provide you with a weekly schedule, a  daily task for the day, a short progress report, a saved reminder and a message reminding you to record your  daily progress.

Option 2 - Allows you to view the entire weekly schedule for Language Learning, so you can view upcoming lessons for the week. The current day and date will be featured at the top of this page for reference.

Option 3 - Leads you to a record progress menu which allows you to record your progress based on the 4 individual Lessons and exercises. The actions to record your progress are fairly simple. select the lesson/exercise you want to record progress for and enter the amount of words learned or exercises/lessons completed. Press enter and the amount will be saved for future reference. Each page will feature how many words/lessons/ exercises are already completed thus far.

Option 4. Allows you to view the progress report for all of your Words Learned and Lessons/exercises completed. It will also give you your over all progress as well as any milestones you may have completed. Milestones are activated when you complete 25%, 50%, 75%, and 100%  of the goal.


Please note that progress values are stored and reused when you run the app again.

## Half Marathon Training

From the Main Menu you are led to this Half Marathon Training menu where you can:

1. View Half Marathon training homepage
2. View 16 week training schedule 
3. Record progress
4. View a progress report
5. Set A Race Date

Before you do anything in this module, you must **set the Race date** which is option 5
The race date ensures the program runs smoothly and all the fields populate correctly. So please do this first before you enter into any of the other menu options for Half Marathon Training.

Adding a Race date is fairly simple. Select option 5. You will be taken to the set Race date page. If a race date is already recorded, the date will be listed at the top of the page. If it is not, just enter a new one. If you want to add the date of August 8, 2026. Type 2026-08-08 and press enter. Now the date is saved. Please note the race day is typically on a Saturday. So choose any Saturday that is  no greater than 16 weeks away.

This Half Marathon Training module tracks your 16 week half marathon training journey. Each day it will tell you what you should do to train for the upcoming half marathon. 

Option 1 leads you to the Half marathon Training Homepage. Over here you will be provided with a weekly schedule,  the  daily task for the day, a short progress report, a reminder if there is one saved and a message reminding you to log your daily progress.

Option 2 Record Progress will lead you to a page where you can enter in your progress. For some this may seem a little confusing seeing that your running schedule is not in front of you. So to ease this confusion, the current weeks runs  and  target miles will be featured at the top of the page for reference. 

The first prompt will ask you to enter a week to record your progress under. Next it will ask you how many runs you are entering in for that week. You can only enter in up to 4 runs for that week. If you don't know if you entered in a run it will tell you how many runs you recorded thus far. Enter your amount and press enter. Next the system will provide you with that weeks schedule and it will ask you how many miles you ran. Add up the total miles for the runs you selected and press enter. If you aren't able to add efficiently, I would suggest that you only enter in 1 run at a time to make this process a little easier. Once you are done you will be taken back to the main menu.

Please note, that if a user is prompted to record progress and does not want to, just press enter to return to the previous menu. No progress will be recorded.

## Novel Creation

From the Main Menu you are led to this Novel Creation Menu  where you can:

1. View novel creation homepage
2. Record 1 draft word count or phase progress
3. View Progess Report

As stated before the Novel Creation Homepage is a place where you can view  the  daily task for the day, access your progress report,  see your reminder if there is one saved, view which novel phase are completed and which ones are pending and a Message reminding you to enter in your  daily progress.

Option 2 allow you to record your progress. The first prompt will ask you for a first draft word count. If you dont have any words to count press 0. Next you will be asked which phase you are completing. just enter a number and press enter. Please not that if you try to enter a phase higher than 2 and your word count is less than 80000 you will not be permitted to do so. Part of phase 2 is writing a first draft that is at least 80k words so please complete that first. Please note, that if a user is prompted to record progress and does not want to, just press enter to return to the previous menu. No progress will be recorded.

Lastly, option 3 allows you to see how many words you recorded and how many phases you completed.

## Journal

Journal options let you:

1. Create Journal Entry
2. View Journal Entries

How journal entry input works:

1. Enter one or more lines.
2. Submit a blank line to finish.
3. Entry is saved to Journal.txt.

The journal is a place to reflect challenges and achievements
Entries are saved so you can view them later on. Again if you dont want to type anything just press enter a few times. to return to the previous menu and nothing will be recorded.

## Set A Reminder

Reminder options are goal-specific:

1. Language Learning reminder
2. Half Marathon reminder
3. Novel Creation reminder

How reminder input works:

1. Existing reminder is displayed first.
2. Enter a reminders and press enter
3. Enter another reminder or Submit a blank line to finish.
4. Reminder file is updated.

Please note that once you type in a new reminder, the old one will be erased. If you intend on keeping that reminder, you must retype it.

## Data Files

Your progress and notes are stored in local files:

- progressMetrics.json
- Journal.txt
- LanguageLearningReminders.txt
- HalfMarathonReminders.txt
- NovelCreationReminders.txt

## Tips For Best Use

1. Update progress immediately after finishing daily tasks.
2. Keep reminders short and specific.
3. Use the journal for blockers, wins, and weekly reflections.
4. Review progress reports weekly.

## Common Issues

1. Nothing saves:
- Ensure you have write permission in the project folder.

2. Unexpected menu behavior:
- Enter only valid menu numbers shown on screen.

3. Lost context across runs:
- Confirm you are launching from the same folder where data files exist.
